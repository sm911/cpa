📚 Terms Explained

Forever-Forward Incremental (FFI)
Take one full backup at the start. After that, take incremental backups every day (only the changes). The chain is maintained so the newest point is always usable without doing fresh fulls each week.
Synthetic Full
A new full backup built from the backups we already have (on the repository). It avoids re-reading all production data and keeps storage use efficient.
Virtual Deduplication Repository
Backup software that finds and removes duplicate blocks across many servers and many days, so we store much less data. It runs as a VM on the IC4G cluster.

How It's Used:
Veeam writes backups to this dedup repository first (global block-level deduplication). Older restore points are then moved to IBM COS (with Object Lock) via SOBR. In practice this provides ~8–12:1 total reduction on mixed VM workloads, which keeps COS ≤ 300 TB while maintaining the 36-day daily schedule.

🛠️ What IBM Consulting Will Configure

Deploy one virtual deduplication repository VM on the existing IC4G hosts.
Deploy one Linux XFS repository VM (fast-clone enabled).
Build SOBR: Performance Tier = {Dedup repo + XFS}; Capacity Tier = COS with Object Lock; policy = Move after N days.
Set Veeam jobs to FFI, compression on, inline dedup on, CBT on, with common exclusions (pagefile/swap/temp/cache) and grouping of similar VMs.
