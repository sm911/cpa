What the System Does

  1. Day 1: Take a full backup from the production systems and land it on the virtual dedupe repository running in IC4G (not on-prem).
  2. Days 2‚Äì36: Take daily changes only and write them to the same dedupe repository; XFS is used as a space-saving work area to build "full" points without copying all data.
  3. After N days (e.g., 7): Automatically move older restore points to IBM Cloud Object Storage (COS) where they are write-protected (Object Lock).
  4. At Day 36: Anything older than 36 days expires per policy.

How to Read the Lifecycle

  • Most recent N days stay "warm" on the IC4G performance tier for quick restores.
  • As each day ages past N, that day's restore point is moved to COS and made immutable.
  • At Day 36, anything older than 36 days expires per the rule.

Component Descriptions

Customer Site ‚Äî Production Systems

  • Generates the data to protect
  • Day 1 captures a full backup
  • Every day after captures only what changed since yesterday

IC4G ‚Äî VCF Tenant (Backup Stack)

  • Veeam Server holds rules for how long to keep and when to move old backups
  • Veeam Proxies pull the data (full on Day 1; daily changes after)
  • Virtual Dedupe Repository stores only unique pieces of data, so repeated data doesn't take extra space
  • XFS Repository is a workspace that can build a "new full" using pointers instead of recopying data
  • SOBR Policy automatically moves older backups to the cloud after N days

IBM Cloud Object Storage ‚Äî Capacity Tier

  • Receives already-reduced backups from IC4G
  • Object Lock makes backups write-protected until their time is up
  • Goal is ‚â§ 300 TB stored after 36 days; this is achieved by deduplicating before sending data to COS

Why This Meets the Target

  • Most duplication is removed in IC4G (virtual dedupe repo), so COS receives much smaller backups.
  • XFS fast-clone avoids large copies when creating "full" points.
  • Move after N days shifts only aged restore points to COS, keeping local restores fast and COS usage predictable.
  • With typical daily change (example ‚âà60 TB) and effective reduction (‚âà10:1), the amount moved per day is about ‚âà6 TB‚Äîaligning the 36-day total to ‚â§ 300 TB.
